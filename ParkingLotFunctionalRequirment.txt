Design Patterns to Apply
Your project already uses SOLID principles. Key patterns for a parking system include: [dev.to]
 
Pattern	Where to Apply
Strategy Pattern	PricingStrategy — Hourly, Flat Rate, Dynamic pricing
Factory Pattern	VehicleFactory — Create Car, Bike, Truck objects
Observer Pattern	DisplayBoard — Notify when spots become free/occupied
Singleton Pattern	ParkingLot — Only one parking lot instance
Builder Pattern	ParkingLotBuilder — Construct complex lot configurations
Saga / Outbox Pattern	Distributed transactions across Payment → Ticket → Notification
